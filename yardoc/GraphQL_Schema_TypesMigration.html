Class: GraphQL::Schema::TypesMigration < GraphQL::Schema::Subset You can add this plugin to your schema to see how GraphQL::Schema::Warden and GraphQL::Schema::Subset handle .visible? differently in your schema. This plugin runs the same method on both implementations and raises an error when the results diverge. To fix the error, modify your schema so that both implementations return the same thing. Or, open an issue on GitHub to discuss the difference. This plugin adds overhead to runtime and may cause unexpected crashes don’t use it in production! This plugin adds two keys to context when running:

  • types_migration_running: true - For the Warden which it instantiates, it adds types_migration_warden_running: true. Use those keys to modify your visible? behavior as needed. Also, in a pinch, you can set skip_types_migration_error: true in context to turn off this plugin’s behavior per-query. (In that case, it uses Subset directly.) Examples:

    Adding this plugin

    if !Rails.env.production? use GraphQL::Schema::TypesMigration end Class methods: use Instance methods: call_method_and_compare, equivalent_schema_members?, initialize, loaded_types
